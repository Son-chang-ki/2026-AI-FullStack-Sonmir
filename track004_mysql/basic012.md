■ 진행1. 정규화 1NF ~ 5NF
 STEP1) 정규화?
  1. 관계형 데이터 모델 - 데이터간 관계에 초점을 둔 모델
  2. 이상(Anomaly) - 공간낭비
    - 삽입, 갱신, 삭제 이상
  3. 정규화? 이상을 제거

 STEP2) ERD 핵심구성요소
  1. 객체(Entity) - 테이블, 개념의 정보단위
  2. 속성(Attribute) - 테이블의 열, 가장 작은 논리적 단위
  3. 관계(Relationship) - 외래키, 테이블 간에 관계를 외래키 등을 구현해서 사용

  예) emp ( FK : dertno ) - dept( PK : deptno )

 STEP2) 2NF - 종속관계
   ~ 알면     ~ 를 알수 있다 
  X(결정자) => Y(종속자)

  1. 기본키가 아닌 모든 속성이 기본키에 완전 종속되면 제2정규형

  X(결정자)              →      Y(종속자)
    1. {STD_ID, 과목}             성적
    2. {STD_ID}                   학부
    3. {STD_ID}                 PROFESSOR
    4. {PROFESSOR}                학부
